Things to keep in mind:

  • *Beta* Build a Burndown Chart!
    • Go to options, select burndown chart checkbox, save.
    • When you build a VMS board, click the icon in the lower left.
    • The burndown currently assumes a two week iteration.
    • The dates are set based on the planned for field starting with the oldest and going to the newest, plus two weeks.
  • *Beta* Go fullscreen in RRC!
    • Go to options, select RRC Fullscreen-inator checkbox, save.
    • When you are in an RRC card, the 'vms' icon should switch to a fullscreen icon.
    • Click the icon to enter in selection mode. Click the icon again to turn it off.
    • In selection mode, the yellow highlighted portion is what will go full screen when you click it.
    • Use the browser's full screen mode for more screen real estate (F11).
  • Bookmarking the VMS Board!
    • The URL will now modify the URL by adding &buildVMS when you build the VMS Board.
    • It will also be removed when you close the VMS Board.
    • If you bookmark and/or navigate to the &buildVMS version of the URL, the page will auto-build the VMS board when you go to it!
  • Keyboard Shortcuts!
    • ctrl + alt + d => Show dependencies for all cards.
    • ctrl + alt + p => Show percentages in column headers.
  • Start typing to filter the cards!
    • The entire text of the visible RTC VMS Board card is searched.
    • Press Escape to clear the filter!
  • RTC/jazz.net fields used (* indicates required):
    • *Id -> Story Number
    • *Summary
    • *Owned By
    • *Status -> Determines the column
    • *Story Points
    • Blocked -> If blocked, display stop sign
    • Depends On -> Shows an orange indicator highlighting that it is dependant on a card
    • Blocks -> Shows a blue indicator highlighting that it blocks a card
    • Planned For -> Used to show iteration the work was planned for Note: Also used in the burndown chart's date logic
    • Resolution Date -> Used to populate data points in the burndown chart
